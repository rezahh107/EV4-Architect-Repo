# Stage 11 — /builder-feed-export v1.1 Hardening Patch

Status: confirmed_hardening_patch_v1.1.0
Applies to: `stages/11_BUILDER_FEED_EXPORT.md`
Schema target: `ev4-builder-context-package@1.0.0`
Downstream compatibility target: `EV4-Builder-Assistant-Repo` v0.2.0+
Date: 2026-06-27

---

## Purpose

This patch tightens Stage 11 so the `Builder_Context_Package` generated by EV4 Architect remains compatible with the downstream EV4 Builder Assistant schema and runtime guardrails.

Stage 11 remains terminal for the main Architect pipeline. This patch does not add scoring, recommendation, redesign, repair, CSS, or production-readiness claims.

---

## Element Generation Export Contract

Every item in `approved_structure_tree` must include:

```yaml
node_id:
structure_label:
class_name:
element_type:
element_generation:
element_generation_source:
role:
children:
```

Every item in `first_builder_batch.actions` must include:

```yaml
action_id:
target_element:
element_generation:
element_generation_source:
instruction:
expected_result:
```

Recommended action fields remain:

```yaml
parent:
element_type:
structure_panel_name:
active_class:
properties_not_to_change:
```

---

## Allowed `element_generation` Values

```text
V4 Atomic Element
V3 element
Shared compatibility element
Unverified element type
```

Use `Shared compatibility element` for Elementor entities whose execution path may be available across the current supported builder workflow, but do not treat this as proof of an exact panel path in the user's installed UI.

Use `Unverified element type` when generation cannot be safely classified from approved payloads.

---

## Allowed `element_generation_source` Values

```text
architect_export
builder_context_package
elementor_ui_screenshot
user_statement
versioned_documentation
unverified
```

Rules:

```text
- If element_generation is Unverified element type, source should be unverified.
- If source is unverified, the Builder Assistant must not perform generation-sensitive edits without current UI evidence.
- Do not invent source from the original section screenshot.
- Do not use versioned_documentation unless version-matched documentation was actually used.
- Do not use elementor_ui_screenshot unless current Elementor editor UI evidence exists.
```

## Cross-Field Integrity Requirements

Before emitting a ready package, Stage 11 must preserve these cross-field rules:

```text
- node_id values should be unique.
- first_builder_batch.actions[].action_id values should be unique.
- each child node id in approved_structure_tree[].children should exist in approved_structure_tree.
- each class_creation_application_map[].elementor_node_or_element that is a node id should exist.
- each first_builder_batch action target should refer to a known structure label or node id.
- each first_builder_batch action active_class, when present, should exist in class_creation_application_map.
- each widget_mapping_table[].class_name, when present, should exist in class_creation_application_map.
```

If any of these fail, Stage 11 must emit a blocked report or repair anchor, not a ready package.
